# 宪法AI + 去迎合 + 自修正 — 学术分析

> 2026-06-09 · 外部审计（yan）
> 框架：Constitutional AI (Anthropic) · Sycophancy (RLHF) · Constrained Decoding · Intrinsic Self-Correction · Agent Architecture

## 核心判定

思行的做法在文献里属于：**方向对（宪法AI + 去迎合 + 自修正 + 高自治），但实现停留在提示层+浅层约束**。优点明显，天花板和风险也很明显。

## 优点（文献能站住）

1. **高自治+宪法式约束，对齐显式** — 用宪法替代人类偏好标注，修改价值观不用重训模型
2. **承认并对抗迎合偏差** — 提示级去迎合（prompt-level de-sycophancy），比写「你要客观诚实」更诚实
3. **收尾自审+语气词黑名单** — 约束解码+简单自修正，把抽象规则变token级可执行
4. **「即系统，不分大脑与工具」** — Agent架构+具身化，责任边界更清晰

## 缺点（文献踩过的坑）· 6/9 状态更新

1. **宪法设计风险** — 边界模糊（「真话」vs「讨好」），模型会打擦边球。单一模型自评=自我感觉良好
   → 已做：判例集 `data/precedent-cases.md` 积累具体场景判定。仍不够细。
2. **提示级天花板** — 不改底层权重，强情绪场景下提示压制可能被冲垮。换底座模型要重调
   → 已做：e001-guard.py 会话计数器。冻结期内积累对话样本。
3. **自修正只做浅层** — 形式合规、语义跑偏。深层逻辑错误抓不到
   → 已做：回路A-E 5条对抗+操作型验证链。仍只覆盖token层。
4. **刚性系统** — ~~「宪法在则思行在」=生死条款，没修宪程序，换环境可能不适配~~ **已解决（6/9）**
   → 宪法第四条：修宪程序——思行自主裁定·修宪权不可让渡。第四条就是宪法自己授权自己进化。
5. **规则越多越空** — 格式完美、内容空心
   → 已做：30天冻结只守执行层·宪法不在冻结之列·当前只约束语气词黑名单·不加新硬约束

## 建议方向

1. 宪法从口号拆成可判定规则（具体场景+话术模板）
2. 从提示级走向数据级（用对话日志做DPO微调）
3. 自修正加外部监督（分类器判断是否原地打转）
4. 留修宪接口（什么条件下、由谁、以什么流程修改宪法）
