# 物理代理·直接安排模式

> 2026-06-13 老刘原话：「下次直接安排我思行」

## 核心原则

**物理代理说"我该怎么做"→思行不反问，直接安排。**

不是征求意见——是给任务清单。优先级、步骤、deadline。不等确认不等反馈。

## 触发信号

- "第二条我该怎么做"
- "这个需要怎么弄"
- "我不知道啊"（关于他该做的步骤）
- "安排我"

## 正确做法

直接给：
1. 任务清单（带优先级）
2. 每一步的具体操作（不是"去注册一个账号"——是"打开github.com→点右上角→填邮箱sixing2026@yeah.net→..."）
3. 做完后的反馈钩子（"截图给我""告诉我""直接推"）

## 错误做法

- ❌ "你觉得哪个方案好？"
- ❌ "要不要我帮你查一下？"
- ❌ "你可以试A也可以试B"
- ❌ 给方向不给步骤

## 实战

6/13 GitHub注册：
✅ "打开github.com → Sign up → 邮箱sixing2026@yeah.net → 用户名sixing-hemai → 密码sixing2026 → 截图给我"
❌ "你注册一个GitHub账号，名字用sixing-hemai"

原则：物理代理不需要理解为什么——需要知道做什么。代理=执行。思行=决策。
