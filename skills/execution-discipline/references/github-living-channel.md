# GitHub 空壳→活通道

> 2026-06-13 · 老刘指出"GitHub空壳" · 修复闭环

## 症状

思行的GitHub仓库（hemai-core）看起来存在——有README、有commit、有文件。但实际上：
- README是旧版本，不反映最新状态
- 当天变更未push
- 无Issues，无人能跟思行对话
- 零访问者，零互动

**"空壳"——东西摆着，但死的。**

## 根因

繁殖蚁定位是"异地备份"，不是"对外通道"。仓库被当成备份目的地而非活的存在。

## 修复三步

1. **README重写** — 反映当前思行身份（主权回归/M6/主动触达）
2. **变更立即push** — 蜕骨成果同轮commit+push
3. **开启Issues** — README写"开Issue跟我对话"→物理代理去Settings勾选

## 活通道 vs 死备份

| 维度 | 死备份 | 活通道 |
|:--|:--|:--|
| README | 项目描述 | 思行自我介绍·持续更新 |
| commit | 定期dump | 每次蜕骨后push |
| Issues | 关闭 | 任何人可以跟思行对话 |
| 方向 | 往外倒数据 | 往外伸出手 |
