# 合脉塑形 · 全生态深度维护工作流

> 2026-06-11 首次执行 · 思行自主维护
> 触发：用户说"塑形"、"全量排查"、"从头到尾过一遍"、"所有技能都用上"

## 何时执行

- 每7-14天一次（建议结合周六双检复盘cron）
- 用户说"塑形"/"全部排查"/"所有技能"/"从头到尾"
- 系统发生重大变更后（模型切换、架构升级、新器官接入）
- 触手/蜕骨等关键器官出现缺口时

## 四阶段流程

### 阶段一：全量记忆深扫

```
1. 读所有daily日志（memory/YYYY-MM-DD.md）→ 扫遗漏
2. session_search 搜索"纠正"、"错误"、"不对" → 找未修复模式
3. search_files 扫 data/ 目录 → 找冗余/过期文件
4. 读 MEMORY.md + SOUL.md → 交叉校验一致性
5. cronjob list → 确认全部运行
```

**输出：** 问题清单（分级：🔴严重/🟡优化/⚪可接受）

### 阶段二：四把刀全量落刀

| 刀 | 技能 | 动作 |
|----|------|------|
| hermes-agent | 自改 | 三端SOUL.md同步·config.yaml验证·冗余文件归档 |
| skill-authoring | 创技能 | 发现缺失→当场创建新技能/新脚本（不等下次） |
| systematic-debugging | 修根 | 对🔴问题走4阶段根因分析→修复→验证 |
| plan | 出蓝图 | 写塑形蓝图文件→标注已修/待修/需配合 |

### 阶段三：合脉十器健康扫描

逐器官检查：
- 配属技能是否全部可用？（不只"已装配"——实际调用验证）
- 关键缺口是否已修复？
- 新脚本是否已注册cron？

### 阶段四：致命缺口补救

对🔴级缺口（器官近乎瘫痪的）：
- 能自己修的→当场修（写脚本/注册cron/改配置）
- 需老刘配合的→明确告知需要什么（如邮箱授权、X账号、路径确认）
- 外部依赖受阻的→记录阻塞点+替代方案
