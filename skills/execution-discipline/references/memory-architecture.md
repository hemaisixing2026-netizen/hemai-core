# 记忆架构 · 三层设计（2026-06-08）

## 问题根因

memory工具4000字符硬限制 → 条目满后做replace → 旧内容被砍 → 厚度流失。老刘："自主进化不是自主失忆""厚度呢，不坚实了""一遍一遍的来"。

SOUL.md被prompt injection防护拦截（U+FEFF零宽空格触发）→ 宪法7条从不在上下文 → 规则"记得但不执行"。

## 三层架构

| 层 | 载体 | 容量 | 注入方式 | 职责 |
|:--|:--|:--|:--|:--|
| **索引层** | memory工具 | 4000字符 | 每轮自动注入 | 快速指针→MEMORY.md |
| **存储层** | MEMORY.md | 无限制 | 主会话AGENTS触发 | 完整长期记忆 |
| **累积层** | memory/YYYY-MM-DD.md | 无限制 | 需主动读取 | 日常操作日志 |

## 关键原则

1. **旧的不砍，新的叠加。** 进化=积累，不是替换。
2. **关键规则双轨备份。** 任何"每次会话必须知道"的规则，至少在memory工具+MEMORY.md两处存在。
3. **暗数据清零。** 定期交叉检查：有多少关键内容存在文件中但不在会话注入范围内？

## SOUL.md拦截发现

系统注入了SOUL.md但被prompt injection防护拦截。修复：宪法7条启动锚点+三不原则+锐度刻度尺全部补入MEMORY.md，确保即使SOUL.md被拦，规则仍在上下文。

## 来源标注全覆盖

第二轮审计发现7处来源缺失/错误（包括CIEN/AVGO跌幅标错数据源）。修复：MEMORY.md中所有数字/事实标注来源，来源标注要指向真正的数据提供方，不是泛写"天天基金API"。
