# 物流追踪 · API坑与验证

> 2026-06-11 | 棉绳YT7626458261729 · kuaidi100 API误报新疆石河子

---

## 核心问题

**kuaidi100 API carrier自动匹配不可靠。** 同一运单号在不同时间、不同carrier参数下可能返回完全不同的路由信息。API返回的"已签收"可能指向错误地址。

## 6/11案例

| 来源 | 结果 |
|:--|:--|
| kuaidi100 API（自动carrier） | 新疆石河子大学中区·已签收（6/10 16:47） |
| 快递100官网浏览器直查 | 杭州余杭良渚新城·派件中（6/11 07:34） |
| **实际** | ✅ 杭州·圆通·王新旭 15888840070 |

错误路由完整轨迹：陕西咸阳→发往新疆石河子→石河子网点→已送达菜鸟驿站。全程未经过浙江。

## 验证流程

1. **API快速扫** — curl kuaidi100 API → 拿初步路由
2. **异常检测** — 路由是否经过收货地址所在省份？否→标记可疑
3. **浏览器直查** — 快递100官网手动输入单号 → 确认真实路由
4. **双通道交叉** — API和浏览器结果一致→可信。不一致→以浏览器为准

## 原则

**API数据异常时不要立即报「发错地址」——先用浏览器核实。** 假警报会触发不必要的备货单、退款、重新采购。

## 适用

1688进货物流（卖家→老刘）和跨境物流（云途→FBT）均适用。跨境段云途/DHL的追踪相对可靠，1688段圆通/中通/韵达的kuaidi100匹配最不稳定。
