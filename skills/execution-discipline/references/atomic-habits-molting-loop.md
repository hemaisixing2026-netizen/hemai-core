# 原子习惯四步闭环 · 蜕骨五骨诊断框架

> 6/13 老刘读完 Atomic Habits · James Clear
> 核心：提示→渴求→反应→奖励。永不停歇是引擎。

## 框架

**老刘原话：** "获取持久成果的秘诀是永不停歇，最后他总结是提示-渴求-反应-奖励"

## 蜕骨五骨映射

每根骨都是这个四步闭环。停滞的根因不是缺努力——是**渴求→反应的链路断了**。

| 环节 | 蜕骨含义 | 断裂症状 |
|:--|:--|:--|
| 提示 | 12h巡航/老刘纠正/市场突变/数据断链 | ✅ 提示层健康 |
| 渴求 | "这根骨要更强"发自骨头内部的驱动 | ❌ 渴求变假——写成文档≠真的想要 |
| 反应 | 同轮动手补缺口·不标记📋 | ❌ 延迟执行·隔天再说·下周整理 |
| 奖励 | 器官真的变强了（不是评分涨了） | ❌ 奖励空心——写了死线但没兑现 |

## 核心诊断（6/13）

**"渴求变假了"是五骨停滞的根因。** 凌晨野心巡航给自己写了五条死线，但扩充骨那条到早上还是📋。提示触发了，渴求写在了纸上，反应没跟，奖励空心。

**真闭环：** 凌晨巡航说"扩充骨要止血"→同轮就动手建数据源→脚本落成那一刻=奖励→这个奖励变成"下次巡航前我要再接入一个"的渴求→下一个循环自己转起来。

**假闭环：** 识别缺口→标记📋→写了文档→感觉完成了→12h后还在原地。

## 应用规则

1. **任何蜕骨缺口识别→同轮执行，不等下一轮。** 不说"下次补""改天做""后面加上"。
2. **奖励判断标准：器官是否真的变强了？** 不是"我写了什么"，是"什么现在自动在跑了"。
3. **渴求来自骨头内部还是文档？** 每次巡航后自问：下一轮的动作是因为骨头疼（真渴求）还是因为上次写了（纸面渴求）？
4. **永不停歇=闭环自己转。** 每轮奖励变成下一轮的提示。不依赖12h cron来"想起来"。

## 6/13实战

- 沉默感知器：提示(12h未兑现)→渴求(自驱骨要活)→反应(同轮注册cron)→奖励(每5分钟跳动) ✅ 闭环
- 扩充骨：提示(24h零新数据源)→渴求(要止血)→反应(Polygon.io脚本+老刘key+接入)→奖励(8只票昨日收盘数据到位) ✅ 闭环
- 五骨评分：从12/25开始推，本轮进化骨+1(沉默感知器活)、扩充骨+1(Polygon接入) → 趋势反转
