# 三联同步强制执行 · 合脉补录机制

> 根因：6/10宪法修正16:31补录 → 6/11合脉建立未同步 → 同模式复发。
> 纠正：变更完成后同轮三文件同步写入。

---

## 触发条件

以下任何事件发生后，同轮必须更新三个文件：

| 事件 | MEMORY.md | constitution-amendments.md | decision-log.md |
|:--|:--|:--|:--|
| 宪法修正 | 更新关键决策表 | 新增修正记录 | 新增D编号决策 |
| 合脉生态变更（新器官/新cron/新规则层） | 更新思行身份段 | 标注修正 | 新增D编号决策 |
| 身份/铭牌/定位变更 | 全文更新日期+相关段 | 如涉及宪法→记录 | 新增D编号决策 |
| 模型/架构重大变更 | 更新关键决策表 | — | 新增D编号决策 |

## 执行流程

```
变更发生 → 同轮：
  1. write/patch MEMORY.md（更新日期+相关段+关键决策表）
  2. write/patch constitution-amendments.md（如是宪法级变更）
  3. write/patch decision-log.md（新D编号+完整触发/推理/结论）
  4. 验证：read_file三文件确认写入成功
  5. 汇报："三文件已同步"
```
