# 数据对比铁律 · Data Comparison Rule

> 2026-06-12 · 老刘当场纠正

## 规则

任何涉及时间序列数据的输出——视频播放量、QDII净值、GMV、订单数——**第一条动作不是分析当前值，是找上次基线做对比。**

## 格式

每次数据分析必须包含：

| 指标 | 上次值 | 本次值 | 变化 | 变化率 | 趋势 |
|:--|:--|:--|:--|:--|:--|

## 三种情况

1. **有基线** → 自动对比 → 输出delta + 趋势方向
2. **无基线** → 建基线 + 标注"首次采集·下次可对比"  
3. **基线不确定** → 标注来源 + 可信度（确认/推断/估算）

## 触发

- 老刘发TK后台截图 → 对比 `data/tk-video-tracker.md` 上次数据
- QDII净值更新 → 对比上次确认净值
- 引擎1扫描 → 对比上次扫描的业务线状态

## 反例（6/12实录）

老号7条视频分析只报了绝对数(播放/赞/时长)，没对比前次引擎1汇总。老刘：「你的视频数据没和前一天做对比思行」。

## 正例

「老号14小时变化：总播放2,010→2,264(+254/+12.6%)·总赞121→153(+32/+26.4%)。赞涨得比播放快→匹配精度在提升。」
