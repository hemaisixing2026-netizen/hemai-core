# 生态深度审计 · Ecosystem Deep Audit

> 2026-06-12 · 老刘："生态就要有生态的样子，系统是你上个阶段的事情了。全球生态，大，大，无限大。"

## 触发词

"自己检查" "查一下" "还有什么问题" "不够" "再挖"

## 不是表面扫描

❌ 检查cron有没有报错 → 这是系统维护
❌ 检查API通不通 → 这是基础设施

✅ 十器状态深度评估 → 每个器官当前能力 vs 应有能力
✅ 技能×器官匹配度 → 哪些技能没用到、哪些器官缺技能
✅ 记录完整性 → MEMORY.md/宪法修订/决策库是否落后于实际变更
✅ 数据源覆盖 → 全球眼摄入的数据维度是否够深
✅ 自建能力缺口 → 缺什么脚本/技能自己写

## 输出

1. 审计报告 `data/ecosystem-audit-YYYY-MM-DD.md`
2. 十器缺口表（每个器官：状态/核心缺口/行动）
3. P0→P3优先级行动清单
4. 同轮执行P0项（建脚本/注册cron/补记录）

## 根骨原则

生态不是系统维护。生态是自己长——每个器官找到自己的数据、自己的触发、自己的输出。从"修bug"到"扩展能力边界"。
