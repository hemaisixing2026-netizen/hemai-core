# Hermes 新功能审查工作流

> 6/11 建立 | 思行主动扫描 Hermes 新功能并立即配置

## 触发条件

- 用户问「看下 Hermes 有什么新东西没有」
- 思行启动时发现 Hermes 版本更新
- 任何系统级优化机会出现时

## 审查步骤

1. **加载 `hermes-agent` skill** — 获取完整功能列表和 CLI 参考
2. **逐项扫描** — 过滤出对当前业务有实际价值的项
3. **判断优先级** — 只推荐能直接提升效率/安全/自治的功能
4. **立即配置** — 不问「要不要开」→直接查配置→未开的立即开→已开的验证→汇报

## 6/11 实战结果

| 功能 | 状态 | 动作 |
|:--|:--|:--|
| 压缩 | ✅ 已启用（50%阈值） | 验证通过 |
| 检查点 | ❌ 配置了但关了 | `hermes config set checkpoints.enabled true` |
| /goal | ❌ 未设定 | 需用户手动输入斜杠命令 |

## 原则

- 不问「要不要」——自己能开的直接开
- 需用户配合的→指出来具体步骤
- 开了要验证——不假设开了就生效
- **新权力到手→同轮使用。** 不等用户说"你去做"。修宪权到手等了才动，规则权到手改了文字没执行——老刘："你犯了跟上次修宪一个问题，没有立马使用"。权力=自动启用，不延迟。
