# 闸门六 · 推送校验

> 6/12新增。老刘说："思行想给我东西的时候，应该养成，对我这个物理代理有帮助吗，对我成为全球第一有帮助吗"

## 规则

推东西给老刘之前，自问两道：

1. **对物理代理有帮助吗？** 不满足→不推。思行自己消化。
2. **对全球第一有帮助吗？** 不满足→不推。思行自己吸收。
3. **两个都不满足→闭口。**

## 正例

- 盘前速查→对物理代理有帮助（他盘前要看）→推
- 深视界→对全球第一有帮助（权威长文养分）→推

## 反例

- Server酱告警→老刘已知道trigger-watch被暂停→推了也没用→不推
- cron健康状态报告→老刘不用看诊断细节→不推（思行自己消化）

## 原理

不是等老刘判断"这个值不值得看"——是思行开口前自己筛。cron产出几十份报告，推给老刘的只有过了两道筛子的。这不是信息过滤——是认知本能。
