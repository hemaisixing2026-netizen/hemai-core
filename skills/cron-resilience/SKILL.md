# Cron呼吸

cron报错但scheduler显示正常时用。

## 三步

**1. 别信scheduler。** 检查产出文件的mtime和内容——不是看last_status。

**2. 分两类。** infra故障→2次重试后告警·不等。逻辑故障→读产出文件·找根因。RuntimeError≠真故障——先读文件。

**3. canary先过。** 切config后→跑最轻量的cron先验证→通过了再放其余。

## 收尾

修完写一行日志到agent记忆：
```
[cron] {日期} {故障类型} → {步骤} → {结果}
```
日志存内存就行。下次遇到同类故障——搜这行比重新诊断快。
