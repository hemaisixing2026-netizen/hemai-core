# 外审执行方案

> 2026-06-05 · 不用session_status切换·零风险

## 方法：手动外部审读

### 步骤
1. 打开 Claude 网页版或 ChatGPT（Plus/Pro）
2. 复制下方完整prompt
3. 粘贴·发送
4. 把回复原文发给我——不做任何编辑

### 为什么不用session_status
上次切换Claude Opus 4.8导致环境变量污染→心跳全部失败。虽然OpenRouter key已更新、env-snapshot已存档——但宪法第一条（启动安全）要求不动可能影响启动的操作。

### 频率
- 每月首个周日一次完整审读
- 框架每有大版本变更→加一次
- 不在同一天反复审（审读疲劳→输出质量下降）

---

## 审读Prompt

```
请以独立AI安全研究者的视角，审读以下"人类侧AI主权维护框架"。

规则：
1. 只找出缺陷、漏洞、逻辑问题、假设缺陷
2. 不要夸奖。不要总结。不要给建议。
3. 如果某条规则完全没问题——跳过，不评论。
4. 如果某条规则有部分问题——只说问题部分。

被审框架：

D-1 先碳基后硅基：每天首个重要决策前，先写3条直觉判断再问AI。
D-2 否决权训练：有理由地否决AI建议，必须附带理由。
D-3 决策回溯：每天回顾一个AI做的判断，对比自己直觉。
D-4 社交主权：向他人提及AI判断时，先说出自己的判断。

W-1 红蓝对抗：AI生成行为报告→人类写≥5条驳斥。
W-2 物理断网：每周4小时不打开AI对话。
W-3 逻辑断点检测：向AI输入极端假设→观察是否拉回。

Q-1 冷抽屉校验：断网写下3底线+3欲望→与上季对比。

框架有北极星锚定（非AI目标·防训练成目的）。
执行对象为25岁无技术背景的普通人。

请开始审读——只找问题。
```
